A. コードエディタ (IDE基盤)

マルチタブエディタ

Rope データ構造によるテキストバッファを採用。O(log n) の挿入・削除、O(1) のクローン（Undo/Redo用）を実現し、大規模ファイルでも高速に動作します。タブの並べ替え、ピン留め、分割表示に対応。

  • 無制限の Undo / Redo（スナップショットベース）
  • タブのドラッグ&ドロップ並べ替え
  • 変更未保存インジケータ (ドット表示)

シンタックスハイライト

正規表現ベースのハイライトエンジンで、以下の言語をサポートしています。

  • Rust キーワード、マクロ、ライフタイム、属性
  • Python デコレータ、f-string
  • JavaScript / TypeScript — JSX 対応
  • C / C++ プリプロセッサディレクティブ対応
  • TOML / JSON / RON

カラースキームは VS Code Dark+ 互換で統一された #D4D4D4 テキストカラーをベースにしています。

ファイルツリー (エクスプローラ)

遅延読み込み方式でディレクトリを展開し、巨大プロジェクトでも即座に表示されます。

  • Codicon アイコンによるファイルタイプ識別
  • ドラッグ&ドロップによるファイル移動
  • 右クリックコンテキストメニュー（新規ファイル/フォルダ、名前変更、削除）
  • .gitignore パターンによる自動フィルタリング

プロジェクト全体検索

Rayon による並列処理と正規表現エンジンにより、数万ファイルでもミリ秒単位で結果を返します。

  • 正規表現 / 大文字小文字区別 / 単語一致フィルタ
  • ファイルタイプ絞り込み（glob パターン）
  • 結果のインライン置換

ファイル内検索・置換

Ctrl+F でファイル内検索バーを表示。ハイライト表示と Enter / Shift+Enter で前後移動。置換モードでは単一・全置換に対応します。

コード折り畳み

ブレース {} やインデントレベルに基づいてリージョンを折り畳み、大規模ファイルの見通しを改善します。行番号ガター横の三角アイコンをクリック、または Ctrl+Shift+[ で操作できます。

マルチカーソル

Alt+クリックで追加カーソルを配置。Ctrl+D で同一単語の次の出現位置にカーソルを追加。すべてのカーソルに同時に入力・編集が反映されます。

ミニマップ

エディタ右端にファイル全体の縮小プレビューを表示。クリックまたはドラッグで高速スクロール。シンタックスハイライトも反映されます。

B. Git パネル (SourceTree風 6タブ)

libgit2 ベースの完全ネイティブ Git クライアント。外部 Git バイナリ不要で、パフォーマンスとポータビリティを両立します。

Status タブ

変更ファイルをステージ済み / 未ステージに分類表示。ファイル単位またはハンク単位でステージ/アンステージ操作が可能です。

  • 差分のインラインプレビュー (統合diff表示)
  • コミットメッセージ入力 + ワンクリックコミット
  • Amend コミットサポート

History タブ

コミットグラフをブランチ分岐・マージを含めて視覚化。各コミットの diff、変更ファイル一覧、メタ情報を閲覧できます。

  • 著者/日付/メッセージでフィルタリング
  • コミット間の差分比較

Branches タブ

  • ブランチ一覧表示（ローカル + リモート追跡）
  • 新規ブランチ作成
  • ブランチ切替 (checkout)
  • マージ操作 (Fast-Forward / 3-Way)
  • ブランチ削除

Remotes タブ

  • リモートリポジトリの追加 / 削除 / 編集
  • Fetch / Push / Pull 操作
  • リモートブランチの追跡設定

Tags タブ

  • タグ一覧表示
  • Lightweight タグ / Annotated タグの作成
  • タグの削除
  • タグへのチェックアウト

Stash タブ

  • 現在の変更をスタッシュに保存 (メッセージ付き)
  • スタッシュの適用 (Apply / Pop)
  • スタッシュの内容プレビュー
  • スタッシュの削除

C. ターミナル

portable-pty による PTY (疑似端末) 統合ターミナル。シェルの完全な対話操作が IDE 内で完結します。

  • フルスクリーンモード切替
  • ANSI カラーコード完全対応
  • シェル自動検出 (zsh / bash / fish)
  • 作業ディレクトリ同期（プロジェクトルート）
  • 複数ターミナルインスタンスのタブ管理
  • Ctrl+` でトグル表示

D. LSP (Language Server Protocol) 統合

stdio ベースのネイティブ LSP クライアントを内蔵。rust-analyzer をはじめとする言語サーバーと通信し、リアルタイムのコード補助を提供します。

  • 自動補完 — タイプ中にリアルタイムで候補を表示。Fuzzy マッチング対応
  • ホバー情報 — シンボル上にカーソルを置くと型情報・ドキュメントをツールチップ表示
  • 診断 (Diagnostics) — エラー・警告を波線で即時表示。問題パネルに一覧集約
  • 定義ジャンプ — Ctrl+クリック または F12 でシンボル定義元へ移動
  • Peek Definition — インラインで定義をプレビュー（ファイル遷移不要）
  • References — シンボルの全参照箇所を検索・一覧表示
  • リネーム — F2 でプロジェクト全体のシンボル名変更

Bevy プロジェクト検出時は rust-analyzer の設定が自動最適化されます (proc-macro 有効化、cargo features 設定など)。

E. AIチャット

berry_api サーバーと gRPC ストリーミングで接続し、LLM によるリアルタイムのコード支援を実現します。

  • ストリーミング応答 — トークン単位でリアルタイム表示
  • コード生成 — 自然言語の指示からコードを生成
  • コード説明 — 選択したコードの解説を生成
  • リファクタリング提案 — 改善案を提示
  • コンテキスト共有 — 現在開いているファイルの内容を自動的にコンテキストとして送信
  • API キーの安全性 — クライアント側に API キーを保存しない設計。すべて berry_api サーバー経由
// gRPC 通信フロー
berrycode (クライアント)
  → gRPC StreamChat Request
    → berry_api (UnifiedLLMService)
      → LLM プロバイダ
    ← gRPC Server-Sent Stream
  ← リアルタイムUI更新

F. Scene Editor — 3Dビューポート

WGPU による GPU オフスクリーンレンダリングで、シーンをリアルタイムに描画。egui テクスチャとして IDE パネル内に表示します。

カメラ操作

操作入力説明
オービットカメラAlt+LMB ドラッグ選択オブジェクトを中心に回転
パンAlt+MMB ドラッグカメラを水平/垂直移動
ズームマウスホイールカメラを前後移動
Fly カメラRMB+WASDFPS スタイルで自由移動
フォーカスF選択オブジェクトにフレーミング

ビュー切替

  • Perspective / Orthographic トグル
  • プリセットビュー: Front / Back / Left / Right / Top / Bottom
  • Quad View (4分割表示) — Top, Front, Right + Perspective を同時表示

ワールドグリッド

XZ 平面に無限グリッドを描画。ズームレベルに応じてグリッド密度が自動調整されます。原点の軸ラインを赤 (X)・緑 (Y)・青 (Z) で色分け表示。

G. Scene Editor — レンダリング品質

ゲームクオリティのリアルタイムレンダリングをエディタ内で実現。各エフェクトは個別に有効化/調整できます。

Bloom高輝度部分のグロー効果。強度 (Intensity) スライダーで調整
Tone Mapping5モード対応: Reinhard / Reinhard Luminance / ACES Fitted / AgX / Blender Filmic
SSAOScreen-Space Ambient Occlusion — 隙間部分の自然な陰影を追加

H. Scene Editor — Transform Gizmo

3D空間でオブジェクトを直感的に操作するためのギズモシステム。

モードショートカット表示操作
Move (移動)W矢印軸方向にドラッグで移動
Rotate (回転)E弧弧をドラッグして回転
Scale (拡縮)R立方体軸方向にスケール変更

平面ドラッグ

軸間の四角パッチをドラッグすると、2軸同時に操作できます (XY / XZ / YZ 平面)。

グリッドスナップ

Ctrl を押しながらドラッグで、設定したグリッド間隔にスナップ。移動 = 1.0 単位、回転 = 15度、スケール = 0.1 単位がデフォルト。

複数選択一括操作

複数エンティティを選択した状態でギズモを操作すると、すべてのオブジェクトに同時に変換が適用されます。ピボットポイントは選択中心。

GPU レイキャスティングにより、メッシュ形状に正確にヒットテストを行います。

J. Scene Editor — Hierarchy パネル

シーン内のすべてのエンティティをツリー構造で表示・管理するパネルです。

ツリー表示

親子関係をインデントで視覚化。折り畳み可能なノードで階層を効率的にナビゲートできます。

ドラッグ&ドロップ Reparenting

エンティティをドラッグして別のエンティティ上にドロップすると、その子要素として再配置されます。ルートレベルへの移動も対応。

フィルタ検索

上部の検索バーにテキストを入力すると、名前が一致するエンティティのみを表示します。

インラインリネーム

エンティティ名をダブルクリックするか F2 を押して、その場で名前を変更できます。

コンテキストメニュー

右クリックで以下の操作が可能:

  • Copy — エンティティをクリップボードにコピー
  • Paste — コピーしたエンティティを貼り付け
  • Duplicate — 選択エンティティを複製
  • Delete — 選択エンティティを削除
  • Save as Prefab — エンティティを .prefab ファイルとして保存

表示/非表示トグル

エンティティ横の目のアイコンをクリックして、ビューポートでの表示/非表示を切り替えます。非表示エンティティはグレーアウト表示。

シーンタブ

複数の .bscene ファイルを同時に開き、タブで切り替えることができます。

K. Scene Editor — Inspector パネル

選択中エンティティのプロパティをリアルタイムに編集するパネルです。

Transform 編集

Position (位置)、Rotation (回転)、Scale (拡縮) を数値入力で編集。Local / World 座標系の切替が可能です。

Add Component

「+ Add Component」ボタンをクリックすると、全29種のコンポーネントから検索フィルタ付きで追加できます。

コンポーネント操作

  • ドラッグで並び替え
  • 右クリックメニューでコピー/ペースト/削除
  • 各コンポーネントの折り畳み表示

GPU Material Preview

マテリアルコンポーネント選択時に、球体プレビューをリアルタイムGPUレンダリングで表示します。パラメータ変更が即座に反映されます。

Prefab 操作

  • Revert — オーバーライドした値を Prefab 元の値に戻す
  • Apply — 現在の値を Prefab ソースに書き戻す
  • Unpack — Prefab リンクを解除し、通常のエンティティに変換

Debug Inspector

Play Mode 中に有効化すると、ECS コンポーネントの内部状態をリアルタイムでモニタリングできます。

L. Scene Editor — コンポーネント全29種

Inspector の「Add Component」から追加可能なすべてのコンポーネントの詳細です。

メッシュ (4種)

MeshCube

立方体メッシュを生成。サイズ (幅/高さ/奥行き) を個別に設定可能。プロトタイピングに最適。

MeshSphere

球体メッシュを生成。半径とセグメント数 (経度/緯度分割数) を設定してポリゴン密度を調整。

MeshPlane

平面メッシュを生成。幅と奥行きを設定。地面、水面、UIバックグラウンドなどに使用。

MeshFromFile

外部3Dモデルファイル (.gltf / .glb / .obj) を読み込んでメッシュとして使用。Asset Browser からの D&D にも対応。

ライト (3種)

PointLight

全方向に光を放つ点光源。色、強度、減衰半径を設定。室内照明やランプの表現に使用。

SpotLight

円錐状に光を放つスポットライト。内角/外角、色、強度を設定。舞台照明やフラッシュライトの表現に。

DirectionalLight

平行光源。太陽光のシミュレーション。方向は Transform の回転で制御。シャドウマッピングの主要光源。

カメラ & オーディオ (3種)

Camera

レンダリングカメラ。Perspective (透視投影) / Orthographic (正射影) 切替。FOV、Near/Far Clip、Priority を設定。

AudioSource

サウンドの発生源。オーディオファイルを指定し、ボリューム、ピッチ、ループ、空間減衰を設定。

AudioListener

音声の受信点。通常はメインカメラにアタッチ。3D空間オーディオの基準位置として機能。

物理 (2種)

RigidBody

物理シミュレーション対象。Dynamic (動的) / Static (静的) / Kinematic (キネマティック) のタイプ選択。質量、摩擦、反発係数を設定。

Collider

衝突判定の形状を定義。Box / Sphere / Capsule / Mesh から選択。Is Trigger フラグで貫通イベント用途にも。

UI (3種)

UiText

テキスト表示。フォント、サイズ、色、整列を設定。ゲーム内のスコア表示やダイアログテキストに使用。

UiButton

インタラクティブなボタン。通常/ホバー/押下の3状態で色/画像を切替。クリックイベントをスクリプトにバインド。

UiImage

画像表示。テクスチャアセットを指定し、色の乗算、UV Rect によるスプライトシート対応。

エフェクト & アニメーション (4種)

ParticleEmitter

パーティクルシステム。放出速度、寿命、サイズカーブ、色グラデーション、重力影響を設定。煙、火、魔法エフェクトに。

Animation

アニメーションクリップのデータ。Position / Rotation / Scale のキーフレームを保持。8種のイージングカーブ対応。

Animator

アニメーションの再生制御。FSM (有限状態機械) によるステート遷移を管理。ブレンド、クロスフェード対応。

SkinnedMesh

ボーンベースのスケルタルメッシュ。頂点ウェイトによるスキニング変形。キャラクターアニメーションの基盤。

スクリプト (2種)

CustomScript

Rust スクリプトファイルをアタッチ。Bevy の System として登録され、ECS クエリでコンポーネントにアクセス。Hot Reload 対応。

VisualScript

ノードベースのビジュアルスクリプト。コーディング不要でゲームロジックを構築。10種のノードタイプを接続して動作を定義。

環境 & 地形 (4種)

Skybox

6面キューブマップまたは HDR 環境マップによる空の表現。IBL (Image-Based Lighting) のソースとしても使用。

Terrain

ハイトマップベースの地形。ブラシツール (Raise / Lower / Smooth / Flatten) でリアルタイム編集。自動LOD対応。

NavMesh

ナビゲーションメッシュ。グリッドベースのベイク処理で歩行可能領域を自動計算。A* パスファインディング対応。

Spline

Catmull-Rom スプラインカーブ。制御点の追加/移動でパスを定義。カメラレール、道路、河川の作成に。

最適化 & その他 (4種)

LodGroup

Level of Detail 制御。カメラ距離に応じて3段階のメッシュを自動切替し、描画パフォーマンスを最適化。

Material (暗黙)

PBR マテリアル。Base Color、Metallic、Roughness、Normal Map、Emissive を設定。Inspector 内で球体プレビュー。

Transform (暗黙)

全エンティティに自動付与。Position / Rotation / Scale。Local / World 座標系。階層的な変換伝播。

Name (暗黙)

エンティティの表示名。Hierarchy パネルでの識別に使用。デフォルトは「Entity_N」形式。

M. Scene Editor — アニメーションシステム

マルチトラックアニメーション

Position / Rotation / Scale を独立したトラックで制御。各トラックにキーフレームを配置し、時間経過に沿ったプロパティ変化を定義します。

8種イージングカーブ

カーブ特性用途例
Linear等速機械的な移動
EaseIn加速落下開始
EaseOut減速着地
EaseInOut加速→減速滑らかな移動
Bounceバウンド跳ねるボール
Elastic弾性ばね的動き
Backオーバーシュート誇張表現
Cubic Bezierカスタム細かい調整

Timeline / Dopesheet ビュー

Timeline — カーブエディタ。キーフレーム間の補間カーブを視覚的に編集。ハンドルをドラッグしてベジェ曲線を調整。

Dopesheet — 全トラックのキーフレームをタイムライン上のドットで一覧表示。キーフレームの移動・複製・削除を一括操作。

Animation Events

特定のフレームにイベントを挿入し、アニメーション再生時にコールバック関数を発火します。足音サウンド、エフェクト生成、状態変更などに使用。

Animator Controller (FSM ノードグラフ)

ノードグラフエディタで有限状態機械を構築。各ステートにアニメーションクリップを割り当て、条件付きトランジションで遷移を制御します。

  • パラメータ (Bool / Float / Trigger) による遷移条件
  • クロスフェード期間の設定
  • Any State / Entry / Exit の特殊ステート
  • サブステートマシン (階層化)

N. Scene Editor — Visual Scripting

コーディング不要でゲームロジックを構築するノードベースエディタです。

10種のノードタイプ

ノード色説明

OnUpdate緑毎フレーム実行される起点ノード

ForLoop黄指定回数のループ実行
GetComponent青指定エンティティからコンポーネントデータを取得
SetComponent青コンポーネントのプロパティ値を設定
MathAdd薄緑2つの数値を加算
MathMultiply薄緑2つの数値を乗算
Compare黄2値を比較 (==, !=, <, >, ≤, ≥)
Print橙Console パネルにデバッグメッセージを出力

エディタ操作

  • 右クリック → ノード追加メニュー (検索フィルタ付き)
  • ポートをドラッグして接続線を描画
  • Delete で選択ノード/接続を削除
  • マウスホイールでズーム、MMB ドラッグでパン
  • ノードのグループ化 (色付きフレームでまとめ)

P. Scene Editor — Terrain

ハイトマップベースの地形編集システム。

ハイトマップ

グレースケール画像 (16bit 対応) またはプロシージャルノイズから地形を生成。解像度は 64x64 から 1024x1024 まで選択可能。

ブラシツール

ブラシ説明
Raise地形を隆起させる。ブラシサイズと強度を調整
Lower地形を陥没させる
Smooth周囲の高さの平均に近づけて滑らかにする
Flattenクリック位置の高さに周囲を揃えて平坦にする

すべてのブラシはリアルタイムプレビュー (半透明円表示) とアンドゥ対応。

メッシュ自動生成

ハイトマップの変更に応じてメッシュが自動再生成されます。法線とタンジェントも再計算され、ライティングが正しく反映されます。

Q. Scene Editor — NavMesh

AI エージェントのナビゲーションに使用するナビゲーションメッシュ。

グリッドベイク

シーン内の Static コライダーを解析し、歩行可能な領域をグリッドとして自動計算します。エージェントの半径、高さ、登坂角度を設定パラメータとして指定。

A* パスファインディング

ベイク済み NavMesh 上で A* アルゴリズムによる最短経路探索を実行。スクリプトから navmesh.find_path(start, goal) で経路を取得できます。

  • NavMesh の可視化オーバーレイ (半透明ブルー表示)
  • 歩行不可領域の赤色ハイライト
  • パスのデバッグ表示 (黄色ライン)

R. Scene Editor — Physics

リアルタイム物理シミュレーション。Play Mode で自動的に有効化されます。

機能説明
重力デフォルト (0, -9.81, 0)。グローバル設定で方向と強度を変更可能
AABB 衝突検出ブロードフェーズで Axis-Aligned Bounding Box を使用した高速衝突判定
摩擦コライダー間の摩擦係数。0.0 (氷) 〜 1.0 (ゴム) で設定
反発反発係数 (Restitution)。0.0 (完全非弾性) 〜 1.0 (完全弾性衝突)
  • RigidBody の速度/角速度をスクリプトから制御可能
  • 衝突イベントのコールバック (OnCollisionEnter / Stay / Exit)
  • トリガー判定 (OnTriggerEnter / Stay / Exit)
  • レイキャスト API でゲームロジックからの線分交差判定

S. Scene Editor — Play Mode

エディタ内でゲームを実行し、リアルタイムにテスト・デバッグを行うモードです。

ボタンショートカット動作
PlayCtrl+Pシーンのスナップショットを保存し、ゲーム実行を開始
PauseCtrl+Shift+P実行を一時停止。Inspector で値を変更して再開可能
StepCtrl+Alt+P1フレームだけ進めて再び一時停止
StopCtrl+P (再押)実行を停止し、スナップショットからシーンを復元

スナップショット復元

Play 開始時にシーン全体の状態をメモリに保存。Stop 時に自動復元されるため、Play 中の変更がエディタのシーンデータを破壊することはありません。

T. Scene Editor — Prefab

再利用可能なエンティティテンプレート。一度定義すれば複数シーンで使い回せます。

保存/読み込み

Hierarchy で右クリック → 「Save as Prefab」で .prefab ファイルとして書き出し。Asset Browser からシーンへドラッグ&ドロップで配置。

ドラッグ&ドロップ配置

Asset Browser の Prefab アセットをビューポートにドラッグすると、マウス位置にインスタンスを配置します。

ネスト対応

Prefab の中に別の Prefab を含めることができます (Nested Prefab)。外側の Prefab を更新しても、内側の Prefab は独立して管理されます。

Revert / Apply / Unpack

  • Revert — インスタンスのオーバーライドを元の Prefab 値に戻す
  • Apply — インスタンスの変更を Prefab ソースファイルに反映。全インスタンスに伝播
  • Unpack — Prefab リンクを切断し、独立したエンティティに変換

U. Scene Editor — Asset Pipeline

アセットのインポートから管理までを統合するパイプラインです。

Import Settings (.meta)

各アセットファイルに対応する .meta ファイルでインポート設定を保存。テクスチャフォーマット、圧縮レベル、メッシュ最適化オプションなどを管理。

テクスチャリサイズ + ミップマップ

インポート時に最大解像度の制限、POT (Power of Two) リサイズ、ミップマップチェーンの自動生成を行います。

サムネイル生成

  • テクスチャ — 縮小画像をキャッシュしてブラウザに表示
  • 3Dモデル — GPU オフスクリーンレンダリングでサムネイルを自動生成

依存関係追跡

アセット間の参照関係を自動解析。未使用アセットの検出、参照先の変更時の影響範囲表示、安全な削除確認を実現します。

V. Scene Editor — ツール群

Profiler (FPSグラフ)

リアルタイムFPSグラフ、フレームタイム詳細、CPU/GPU 使用率を表示。ボトルネックの特定に。

Console

ログメッセージの一覧表示パネル。レベル (Info / Warning / Error) でフィルタ、テキスト検索、ログファイルへの保存に対応。

Dockable Tool Panel

すべてのパネルはドラッグでドッキング位置を変更可能。タブ化、フローティング、分割表示に対応し、ワークスペースをカスタマイズできます。

Scene Merge (Three-Way Diff)

バイナリではなく RON テキストベースの .bscene フォーマットにより、Git での Three-Way マージが可能。コンフリクトを視覚的に解決します。

Build Settings + Pipeline

ターゲットプラットフォーム (Windows / macOS / Linux) の選択、ビルド構成 (Debug / Release)、出力先の設定。ワンクリックでバイナリを生成します。

Hot Reload

Rust スクリプトファイルの変更を検出し、Play Mode を停止せずにコードを再読み込み。イテレーション速度を大幅に向上させます。

Spline 描画

ビューポート上で制御点をクリックしてスプラインを作成。カメラパス、レール、道路のレイアウトを視覚的に設計できます。

Skeleton 表示

SkinnedMesh のボーン構造をワイヤーフレームで可視化。ボーンの選択・Transform 編集が可能で、アニメーションのデバッグに使用します。
